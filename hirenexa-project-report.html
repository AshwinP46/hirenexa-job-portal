User Roles & Features
👨‍🎓
Student Portal
Route: /student — Accessed by enrolled placement-seeking students
  • Personalized dashboard with active job count, interview count, applied jobs, shortlists, selections, and rejections.
  • Monthly application activity area chart and status distribution pie chart with live data from Supabase.
  • Smart AI-style resume insights — shows profile strength score, skill gaps, and improvement tips.
  • Profile page with full info editing: name, email, phone, department, CGPA, bio, year of study, and roll number.
  • Profile picture upload — hover over avatar to upload an image (stored in Supabase, shown in header & profile).
  • Resume PDF upload — stored securely in Supabase Storage, accessible via signed URL.
  • Dynamic skill tags — add or remove skill tags from profile with instant save.
  • Job Board with 60+ live jobs — filters by salary (LPA), city, company, and skill tags.
  • AI match score — each job card shows a percentage match computed from student skills vs job requirements.
  • One-click job applications with real-time eligibility check (CGPA filter).
  • Save/Bookmark jobs to view later.
  • Application History timeline — shows all applied jobs with status, company logo, package, and timeline.
  • Interview Calendar — full month view calendar highlighting scheduled interview dates.
  • Upcoming interview list with meeting link, round name, time, and mode (Online/Onsite).
  • All Interviews table with sortable columns including company, role, date, time, mode, and status.
  • Achievement badges — awarded for milestones like first application, shortlist, offer accepted, etc.
  • Notification bell in header with unread count, mark-as-read, and mark-all-read support.
  • Dark mode toggle (persists per session).
🏢
Recruiter Portal
Route: /recruiter — Accessed by company HR teams and hiring managers
  • Recruiter dashboard with live KPIs: Total Applicants, Shortlisted, Interviews Scheduled, and Offers Made.
  • Applicant pipeline manager — view applicants with CGPA, skills, ATS match score, and application date.
  • One-click status update on candidates: Shortlist, Reject, Schedule Interview, or Mark as Selected.
  • Interview scheduler — set date, time, mode (online/onsite), meeting link, and round name.

  • Smart funnel insights card — shows hire conversion rates, avg match score, and action suggestions.
  • Weekly hiring trends bar chart for visualizing application inflow patterns.
  • Company profile page with branding, industry details, and website link.

  • Global analytics dashboard: total students, companies, placements, and average placement package.

Database Schema
🔷public.profiles
Linked to Supabase Auth. Holds user identity data: full name, email, phone, avatar_url (profile picture), and is_demo flag for seeded data.
🔐public.user_roles
Maps user IDs to roles: student, recruiter, or admin. Supports multiple roles. Priority order: admin > recruiter > student for dashboard routing.
👨‍🎓public.students
Extended student data: department, CGPA (decimal), year of study (1–4), roll number, skills[] array, bio text, and resume_url for storage path.
🏢public.recruiters
Company profile: company name, recruiter contact name, industry, website URL, and optional logo_url for displaying company branding.
💼public.jobs
Job listings: title, description, location, package_lpa (numeric), minimum_cgpa, job_type, skills_required[], openings count, application deadline, and is_active flag.
📝public.applications
Links students to jobs. Status enum: applied → shortlisted → interview_scheduled → selected / rejected. Unique constraint on (student_id, job_id). Stores ATS match_score.
📅public.interviews
Interview scheduling: linked to application, interview_date (timestamptz), mode (online/onsite), meeting_link, location, round_name, status, and feedback text.
🔔public.notifications
Real-time alerts: title, message, is_read boolean, linked user_id. Created by triggers on application status changes, interview scheduling, and recruiter actions.
📋public.audit_logs
System-level action tracking: actor user_id, action type (e.g., "shortlisted_candidate"), target entity, and timestamp for compliance and recruiter accountability.

Security Architecture
Row Level Security (RLS) — PostgreSQL

Every database table is protected by granular Postgres Row Level Security policies. No data can be read, written, or updated without passing the relevant user-role check.

  • Students can only read/write their own profile, applications, and interviews.
  • Recruiters can only view and update applications and interviews tied to their own job postings.
  • Admins have full read access across all entities using the has_role(auth.uid(), 'admin') helper.
  • Supabase JWTs are validated on every API request. No server-side session storage required.
  • Storage bucket (resumes) enforces per-user-folder access — students can only access files inside their own UID-prefixed path.
  • All sensitive operations (status updates, interview creation) are wrapped in RLS-enforced INSERT/UPDATE policies.

How to Run the Project
Step Action Command / Instructions
1 Extract ZIP Right-click final-hirenexa.zip → Extract All
2 Install dependencies npm install
3 Start development server npm run dev → opens at http://localhost:5173
4 Load seed data into Supabase Paste contents of supabase/seed.sql into Supabase SQL Editor → Run
5 Create Admin Account Register on the site, then manually set role to "admin" in Supabase → user_roles table
6 Build for production npm run build
